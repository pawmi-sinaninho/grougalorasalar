# ACCEPTANCE GATES

## Phase 1 gate — passed

- domain objects are defined;
- uncertainty is explicit;
- all critical rules have stable IDs;
- evidence capture is specified;
- Codex timing is corrected.

## Phase 2 gate — passed as draft single-reference specification

- logical axes and a reproducible reference transform exist;
- every candidate cell has an explicit authority class;
- the supplied screenshot is manually reconstructable;
- boundary and occlusion uncertainty blocks silent solving;
- editor interactions, audit and validation are specified;
- machine-readable arena and reference-turn files validate.

This gate does not verify the final edge mask or projection anchor.

## Phase 3 gate — passed as implementation-neutral contract

- every fixture declares expected definite and conditional action sets;
- status precedence is exact;
- ranking and canonical tie-breaks are machine-readable;
- hidden mechanics switches were added to `RulesProfile`;
- two compliant implementations can compare canonical signatures and ordered outcomes.

This gate proves specification consistency, not live-game correctness.

## Phase 4 gate — passed as specification

- every critical field has a primary extraction method;
- independent cross-checks are defined where feasible;
- field confidence, hard caps and thresholds are machine-readable;
- every failure has a correction path and solver consequence;
- dataset, annotation, split and regression protocols are fixed;
- false-safe and auto-confirm entry gates are exact;
- 20 visual contract fixtures validate.

This gate proves contract completeness, not detector accuracy. Automatic critical-field confirmation remains disabled.

## Phase 5 gate

The complete pre-live build specification contains no unresolved ownership between frontend, vision and rules services.

## Phase 6 gate

A local pre-live application completes the full screenshot-to-instruction flow.

## Phase 7 gate

Measured results exist on a locked validation corpus. Critical false-safe recommendations are zero on that corpus.

## Phase 8 gate

Codex changes preserve or improve all baseline tests and measured accuracy.
